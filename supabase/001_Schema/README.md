## 1. الأنواع (Enums)

### 🔐 `role_name` - أسماء الأدوار

| القيمة     | الوصف       | الاستخدام                                     |
| ---------- | ----------- | --------------------------------------------- |
| `admin`    | مدير النظام | إدارة المتجر والمنتجات والطلبات والمستخدمين   |
| `vendor`   | بائع/موظف   | إدارة المنتجات والطلبات (موظف في المتجر)      |
| `customer` | عميل        | تصفح المنتجات، تقديم الطلبات، كتابة التقييمات |
| `delivery` | موظف توصيل  | توصيل الطلبات والتحقق من التسليم عبر QR Code  |
| `support`  | دعم فني     | الرد على تذاكر الدعم ومساعدة العملاء          |

**الجداول المستخدمة:** `core_role`, `core_profile_role`

---

### 📦 `order_status` - حالة الطلب

| القيمة             | الوصف       | الاستخدام                   |
| ------------------ | ----------- | --------------------------- |
| `pending`          | في الانتظار | طلب جديد لم تتم معالجته بعد |
| `confirmed`        | مؤكد        | تم تأكيد الطلب من الإدارة   |
| `processing`       | قيد التحضير | يتم تجهيز الطلب حالياً      |
| `out_for_delivery` | في الطريق   | الطلب مع الموظف للتوصيل     |
| `delivered`        | تم التسليم  | ✅ تم تسليم الطلب للعميل    |
| `cancelled`        | ملغي        | ❌ تم إلغاء الطلب           |

**الجداول المستخدمة:** `trade_order`

---

### 🚚 `delivery_status` - حالة التوصيل (QR Code)

| القيمة             | الوصف       | الاستخدام                                          |
| ------------------ | ----------- | -------------------------------------------------- |
| `pending`          | في الانتظار | لم يتم التوصيل بعد                                 |

#### 📋 `core_profile` - الملف الشخصي للمستخدم

| ------------------- | ------------ | ------------------------- | ------------------------------------------- |

| `phone_number`      | varchar(20)  | رقم الهاتف                | للتواصل والتحقق (فريد)                      |
| `is_phone_verified` | boolean      | هل تم التحقق من الهاتف    | true/false للتحقق من صحة الرقم              |
| `deleted_at`        | timestamptz  | تاريخ الحذف (Soft Delete) | null = نشط، تاريخ = محذوف                   |
| `created_at`        | timestamptz  | تاريخ الإنشاء             | متى تم إنشاء الحساب                         |
| `updated_at`        | timestamptz  | تاريخ آخر تحديث           | آخر مرة تم تعديل البيانات                   |

**العلاقات:**

- `auth.users.id` ← `core_profile.id` (حذف CASCADE)

#### 🔑 `core_role` - الأدوار والصلاحيات

**الغاية:** تعريف الأدوار المتاحة في النظام وصلاحياتها

| الحقل         | النوع       | الوصف                                | الاستخدام                                                       |
| ------------- | ----------- | ------------------------------------ | --------------------------------------------------------------- |
| `id`          | uuid        | المعرف الفريد للدور                  | المفتاح الأساسي                                                 |
| `code`        | role_name   | كود الدور (admin, customer, support) | نوع الدور                                                       |
| `description` | text        | وصف الدور                            | شرح ما يفعله هذا الدور                                          |
| `permissions` | jsonb       | قائمة الصلاحيات                      | مصفوفة JSON بالصلاحيات مثل: `["orders:read", "products:write"]` |
| `created_at`  | timestamptz | تاريخ الإنشاء                        | متى تم إنشاء الدور                                              |
| `updated_at`  | timestamptz | تاريخ آخر تحديث                      | آخر تعديل على الدور                                             |

**مثال على الصلاحيات:**

```json
{
  "code": "admin",
  "permissions": ["*:*"]
}
```

```json
{
  "code": "customer",
  "permissions": ["orders:read", "orders:create", "reviews:write"]
}
```

#### 🔗 `core_profile_role` - ربط المستخدم بالأدوار

| ------------- | ----------- | ----------------- | ---------------------- |
| `profile_id`  | uuid        | معرف المستخدم     | FK → core_profile.id   |
| `role_id`     | uuid        | معرف الدور        | FK → core_role.id      |
| `assigned_at` | timestamptz | تاريخ تعيين الدور | متى تم إعطاء هذا الدور |
| `updated_at`  | timestamptz | تاريخ آخر تحديث   | آخر تعديل              |

**ملاحظات:**

- المفتاح الأساسي مركب: `(profile_id, role_id)`
- المستخدم يمكن أن يكون له عدة أدوار

#### 📱 `trade_order_delivery` - تسليم الطلبات (QR Code)

**الغاية:** تتبع تسليم الطلبات والتحقق منها عبر QR Code

| الحقل               | النوع           | الوصف                  | الاستخدام                                       |

| `id`                | uuid            | المعرف الفريد          | المفتاح الأساسي                                 |

| `delivery_status`   | delivery_status | حالة التوصيل           | pending, delivered, failed, returned            |

**سير عمل QR Code:**

```
1. إنشاء الطلب → إنشاء verification_code
2. عرض QR Code في لوحة التحكم
3. الموظف يطبع QR ويضعه على الطلب
4. عند العميل → يمسح QR
5. verification_code يُتحقق منه
6. customer_verified = true
7. delivery_status = "delivered" ✅
```
