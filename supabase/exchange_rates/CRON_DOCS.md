# توثيق مهام Cron - أسعار صرف العملات

## 📋 نظرة عامة

يحتوي هذا الملف على إعدادات مهمة Cron التلقائية لتحديث أسعار صرف العملات في منصة Marketna.

**الملف:** `supabase/exchange_rates/vercel.json`

---

## 🕐 المهمة المُعدّة

### تحديث أسعار الصرف (`update-rates`)

**المسار:** `/api/cron/update-rates`  
**الجدول:** `10 2 * * *`

#### الجدول الزمني

| المنطقة الزمنية | الوقت |
|-----------------|-------|
| UTC | 02:10 صباحاً |
| سوريا (UTC+3) | 05:10 صباحاً |
| السعودية (UTC+3) | 05:10 صباحاً |
| الإمارات (UTC+4) | 06:10 صباحاً |

#### الوصف

تقوم هذه المهمة بتحديث أسعار صرف العملات من مصدر خارجي (ExchangeRate-API) وتخزينها في جدول `exchange_rates`.

**معايير التحديث:**
- جلب الأسعار من `https://v6.exchangerate-api.com/v6/{KEY}/latest/USD`
- تحديث جميع العملات في قاعدة البيانات
- تحديث `last_updated_at` للتوقيت الحالي

## 📋 صيغة Cron Expression

```
┌───────────── الدقيقة (0 - 59)
│ ┌───────────── الساعة (0 - 23) بتوقيت UTC
│ │ ┌───────────── يوم الشهر (1 - 31)
│ │ │ ┌───────────── الشهر (1 - 12)
│ │ │ │ ┌───────────── يوم الأسبوع (0 - 6) حيث 0 = الأحد
│ │ │ │ │
10 2 * * *
```

### التحليل

| الخانة | القيمة | المعنى |
|--------|--------|--------|
| الدقيقة | `10` | عند الدقيقة 10 |
| الساعة | `2` | عند الساعة 02:00 UTC |
| يوم الشهر | `*` | كل يوم |
| الشهر | `*` | كل شهر |
| يوم الأسبوع | `*` | كل يوم أسبوع |

**النتيجة:** **يومياً الساعة 02:10 صباحاً UTC**

## 🔧 كيفية التعديل

### 1. تحديد الوقت المطلوب

```
مثال: أريد التحديث الساعة 03:00 فجراً بتوقيت سوريا

الخطوات:
1. سوريا UTC+3
2. 03:00 سوريا = 00:00 UTC
3. التعبير: "0 0 * * *"
```

### 2. تحديث ملف vercel.json

```json
{
  "crons": [
    {
      "path": "/api/cron/update-rates",
      "schedule": "0 0 * * *"
    }
  ]
}
```

### 3. النشر على Vercel

```bash
git add supabase/exchange_rates/vercel.json
git commit -m "Update exchange rates cron schedule"
git push
```

## ⚠️ ملاحظات هامة

### 1. التوقيت UTC

جميع أوقات Vercel Cron تستخدم **UTC** فقط. يجب تحويل التوقيت المحلي إلى UTC.

### 2. مدة التنفيذ القصوى

| نوع الوظيفة | المدة القصوى |
|-------------|--------------|
| Edge Function | 10 ثواني |
| Serverless Function | 60 ثانية |

### 3. عدد المهام المسموح

| الخطة | عدد مهام Cron |
|-------|---------------|
| Hobby (مجاني) | 1 |
| Pro | 10 |
| Enterprise | غير محدود |

### 4. سبب اختيار 02:10 UTC

- **02:00 UTC** = وقت هادئ (أقل استخدام للنظام)
- **+10 دقائق** = بعد مهام الصيانة الأخرى
- **يومياً** = لضمان تحديث الأسعار

## 🔍 استكشاف الأخطاء

### المشكلة: المهمة لا تعمل

**الحل:**
1. تحقق من أن `CRON_SECRET` موجود في `.env`
2. تحقق من أن `/api/cron/update-rates` موجود
3. تحقق من سجلات Vercel Dashboard

### المشكلة: الأسعار لا تُحدّث

**الحل:**
1. تحقق من أن `EXCHANGE_RATE_API_KEY` صحيح
2. تحقق من أن `SUPABASE_SERVICE_ROLE_KEY` صحيح
3. تحقق من وجود جدول `exchange_rates`

### المشكلة: المهمة تفشل

**الحل:**
```bash
# اختبار المهمة يدوياً
curl -H "Authorization: Bearer $CRON_SECRET" \
  https://your-domain.com/api/cron/update-rates
```
